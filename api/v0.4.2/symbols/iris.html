  • <static> iris.Goto(p_hashUri)
    Navigate to a screen. The screen must be previously added using iris-_AbstractScreen#AddScreen.
    You can send parameters to the target screen as ... /screen?param1=value1¶m2=value2/ ..., remember apply encodeURI() to the parameter values. 
    iris.Goto("#home/section/subsection");

// Call with screen parameters
iris.Goto("#home/section?id=5&name=example%20name/subsection");

// Remember encode param values
var title = encodeURI("Title Example");
iris.Goto("#home/section?title=" + title);
    Parameters:
    p_hashUri
    {String} Hash URL
  • <static> iris.Include(File)
    Include static resources dynamically. The load is synchronous and tracks all the files included to prevent them to be included twice.
    You can include:
    • CSS files: style files are included by creating a link dynamically in the header of the index page.
    • JS files: script files are included by creating a script object dynamically in the header of the index page and append the loaded content to it. When including javascript files you must take care of the syntax as including javascript file with some syntax error would produce an error when adding it to a script element.
    iris.Include("app/service/myservice.js", "app/css/mycss.css");
    Parameters:
    {arguments} File
    paths
